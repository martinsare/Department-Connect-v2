Here's a quick breakdown of each role in Department Connect:

**Student** — The base tier. Can view their class schedule, check live class status, scan QR codes for attendance, see their attendance records, receive notifications, view contribution details, and make payments via Paystack.

**Admin** — Covers Lecturers, Course Representatives, and Department Executives under one tier. Can create and manage student accounts, approve or reject pending registrations, schedule and manage class sessions, update live class status, generate QR attendance, post announcements, create contributions and events, and view department analytics.

**Developer** — The top tier, inherits everything from both Student and Admin. On top of that, they can assign and change user roles, access system configuration and feature flags, query the database directly, view full audit logs of all user actions, recover any account, and act as any other role to debug or test the app.

The key distinction is that **Students consume**, **Admins operate**, and **Developers control**. Each tier can do everything the tier below it can, plus more.
